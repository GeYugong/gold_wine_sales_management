# Gold 红酒销售管理系统 测试用例

本文档为 Gold 红酒销售管理系统的手工测试用例，覆盖六大功能模块及核心业务规则，供功能验证和答辩演示时逐项核对。

## 测试环境与前置说明

- JDK 8 或更高版本，已构建项目。
- 通过 `run.bat` 或 `./mvnw exec:java` 启动控制台程序。
- 除特别说明外，各用例以「重置数据库后的初始种子数据」为前置条件。可执行 `reset-database.bat` 删除根目录 `wine.db`，下次启动自动重建并导入种子数据。
- 初始种子数据：18 条产地、30 条红酒、5 位客户、1 条已完成订单（客户 2 购买红酒 1，数量 2）。

## 用例编号规则

编号格式为 `TC-<模块>-<序号>`，模块缩写如下：

| 缩写 | 模块 |
| --- | --- |
| WINE | 红酒信息管理 |
| ORIG | 产地信息管理 |
| CUST | 客户信息管理 |
| CHKT | 购买结算管理 |
| ORDR | 订单查询管理 |
| STAT | 销售统计管理 |
| VALD | 通用输入校验 |

## 一、红酒信息管理

| 用例编号 | 用例说明 | 操作步骤与输入 | 预期结果 |
| --- | --- | --- | --- |
| TC-WINE-01 | 添加合法红酒 | 主菜单 1 → 1，依次输入名称、品牌、类型、年份(如 2021)、已存在的产地编号、葡萄品种、进货价、销售价、库存、状态选「上架」，来源链接留空 | 提示「保存成功。」，查询全部红酒可见新记录 |
| TC-WINE-02 | 修改红酒信息 | 主菜单 1 → 2，输入已存在红酒编号，修改各字段 | 提示「保存成功。」，再次查询显示修改后的值 |
| TC-WINE-03 | 修改时输入不存在的红酒编号 | 主菜单 1 → 2，输入不存在的编号(如 9999) | 提示「红酒不存在，请重新输入。」，循环要求重新输入 |
| TC-WINE-04 | 删除无订单明细的红酒 | 先添加一条新红酒，主菜单 1 → 3 输入该编号 | 提示「删除成功。」 |
| TC-WINE-05 | 删除存在订单明细的红酒 | 主菜单 1 → 3，输入红酒编号 1（初始订单已引用） | 提示「该红酒存在订单明细，不能删除。」 |
| TC-WINE-06 | 查询全部红酒 | 主菜单 1 → 4 | 以表格形式列出全部红酒，列对齐正常 |
| TC-WINE-07 | 按名称模糊查询 | 主菜单 1 → 5，输入关键词「张裕」 | 仅显示名称含「张裕」的红酒 |
| TC-WINE-08 | 按类型查询 | 主菜单 1 → 6，输入「干红」 | 仅显示类型含「干红」的红酒 |
| TC-WINE-09 | 按产地查询 | 主菜单 1 → 9，输入产地编号 1 | 仅显示 origin_id=1 的红酒 |
| TC-WINE-10 | 上架 / 下架红酒 | 主菜单 1 → 7，输入红酒编号，选新状态「下架」 | 提示「状态已更新。」，查询显示状态为「下架」 |
| TC-WINE-11 | 查询库存不足红酒 | 主菜单 1 → 8 | 显示库存小于 20 的红酒（按库存升序） |
| TC-WINE-12 | 价格校验 | 添加红酒时进货价或销售价输入负数 | 提示金额不能小于 0，要求重新输入 |
| TC-WINE-13 | 库存校验 | 添加红酒时库存输入负数 | 提示请输入不小于 0 的整数 |
| TC-WINE-14 | 年份校验 | 添加红酒时年份输入 1800 或未来年份 | 提示年份必须在 1900 到当前年份之间 |
| TC-WINE-15 | 状态取值校验 | 添加红酒时状态输入「上」等非法值 | 提示只能输入「上架、下架」 |

## 四、购买结算管理

| 用例编号 | 用例说明 | 操作步骤与输入 | 预期结果 |
| --- | --- | --- | --- |
| TC-CHKT-01 | 普通客户下单（无折扣） | 主菜单 4 → 1，选普通客户、上架红酒、数量、支付方式 | 实付金额 = 售价 × 数量，优惠为 0，订单创建成功 |
| TC-CHKT-02 | 品鉴会员下单（95%） | 选品鉴会员（客户 2）购买 | 实付 = 原价 × 0.95，优惠为原价的 5% |

| TC-CHKT-05 | 下单后库存扣减 | 下单前记录库存，下单后再查询该红酒 | 库存 = 原库存 − 购买数量 |

| TC-CHKT-09 | 支付方式校验 | 支付方式输入非法值 | 提示只能输入现金 / 微信 / 支付宝 / 银行卡 |
| TC-CHKT-10 | 取消订单并恢复库存 | 主菜单 4 → 2，输入已完成订单编号 | 提示「订单已取消，库存已恢复。」，库存回到取消前 + 明细数量 |
| TC-CHKT-11 | 重复取消拦截 | 对已取消订单再次执行取消 | 提示「已取消订单不能重复取消。」 |

## 八、端到端演示流程（冒烟测试）

按以下顺序执行，可完整验证核心链路，也适用于答辩演示：

1. 执行 `reset-database.bat` 重置数据库。
2. 启动系统，主菜单 1 → 4 查询全部红酒，记录目标红酒库存。
3. 主菜单 3 → 1 新增一个客户（或选用初始客户）。
4. 主菜单 4 → 1 创建购买订单，确认折扣与实付金额正确。
5. 主菜单 1 → 4 再次查询红酒库存，确认已扣减。
6. 主菜单 5 → 2 查询订单详情，确认订单主表与明细正确。
7. 主菜单 4 → 2 取消该订单，确认状态变为「已取消」。
8. 主菜单 1 → 4 再次查询库存，确认已恢复。
9. 主菜单 6 → 1 查询销售总额，确认已取消订单不计入统计。
10. 主菜单 6 → 4 查询库存不足红酒，确认提醒正常。
