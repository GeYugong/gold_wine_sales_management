# Gold 红酒销售管理系统开发计划

## 一、项目背景

本项目是 Java 高级编程大作业第 26 题：**红酒销售管理系统**。

系统名称：**Gold 红酒销售管理系统**

项目要求使用 **Java 语言** 完成，并且必须是 **控制台程序**。系统需要结合数据库完成数据存取，并采用分层架构设计。

本项目主要实现红酒信息管理、产地管理、客户管理、购买流程管理、费用结算、订单查询和销售统计等功能。

---

## 二、硬性开发要求

1. 使用 Java 语言开发。
2. 只能做控制台程序。
3. 不能使用 JavaWeb、Swing、JavaFX、Android、微信小程序等形式。
4. 必须连接数据库进行数据存取。
5. 必须使用分层架构设计。
6. 代码中必须加入注释。
7. 每个类都要有完整类注释，包含：

   * 编写者
   * 完成时间
   * 类的具体功能
8. 项目需要能正常运行，方便答辩演示。
9. 开发过程中先统一不写注释，优先完成可运行功能；功能稳定后再统一补充类注释和必要代码注释。

## 五、功能详细设计

### 1. 红酒信息管理

#### 主要功能

* 添加红酒
* 修改红酒
* 删除红酒
* 查询全部红酒
* 根据名称模糊查询红酒
* 根据类型查询红酒

* 红酒上架
* 红酒下架
* 查询库存不足红酒

#### 红酒字段设计

红酒信息至少包含：

| 字段             | 说明             |
| -------------- | -------------- |
| id             | 红酒编号           |
| name           | 红酒名称           |
| brand          | 品牌             |
| type           | 类型，例如干红、干白、起泡酒 |
| year           | 年份             |
| origin_id      | 产地编号           |
| purchase_price | 进货价            |
| sale_price     | 销售价            |
| stock          | 库存数量           |
| status         | 状态：上架 / 下架     |

## 八、各层职责说明

### 1. entity 层

负责定义实体类。

例如：

* Wine 表示红酒
* Origin 表示产地
* Customer 表示客户
* SaleOrder 表示订单
* SaleOrderItem 表示订单明细

---

### 2. dao 层

负责数据库操作。

包括：

* 增加数据
* 删除数据
* 修改数据
* 查询数据

要求：

* SQL 语句主要写在 DAO 层。
* 不要把 SQL 写到 UI 层。
* DAO 层不处理复杂业务逻辑。

---

### 3. service 层

负责业务逻辑。

例如：

* 判断库存是否充足
* 计算会员折扣
* 计算订单总价
* 创建订单
* 取消订单
* 恢复库存
* 判断是否允许删除数据

---

### 4. ui 层

负责控制台菜单。

包括：

* 显示菜单
* 接收用户输入
* 调用 service 层方法
* 输出操作结果

---

### 5. util 层

负责工具类。

包括：

* 数据库连接工具
* 输入校验工具
* 日期处理工具

## 十、核心业务规则

1. 添加红酒时，价格不能小于 0。
2. 添加红酒时，库存不能小于 0。
3. 创建订单时，购买数量必须大于 0。
4. 如果库存不足，不能创建订单。
5. 订单创建成功后，必须扣减库存。
6. 取消订单后，必须恢复库存。
7. 已取消订单不能重复取消。
8. 删除客户前，如果客户存在订单，不能删除。
9. 删除红酒前，如果红酒存在订单明细，不能删除。
10. 删除产地前，如果该产地下存在红酒，不能删除。
11. 用户输入错误时，程序不能崩溃，需要提示重新输入。
12. 系统启动时，如果数据库不存在，需要自动创建数据库和数据表。
13. 系统启动时，可以自动插入一些测试数据，方便演示。

## 十一、数据库初始化要求

程序第一次启动时，需要自动完成：

1. 创建数据库文件 `wine.db`
2. 创建所有数据表
3. 插入基础测试数据

测试数据建议包括：

* 3 个国内产地
* 3 个国外产地
* 10 条红酒数据
* 5 个客户数据
* 若干订单数据

项目根目录中的 `gold_wine_seed_data/` 为当前提供的原始项目数据，后续数据库初始化、红酒基础数据和产地基础数据应优先参考该目录中的数据文件。

该目录目前作为临时原始数据目录保留在根目录。项目结构确定后，需要将其中的数据移动到仓库中更合适的位置，例如：

* `src/main/resources/data/`：用于程序运行时读取的初始化数据
* `docs/data/`：用于文档说明、数据来源记录和人工整理材料

移动后需要同步更新代码、README 或相关说明，避免继续依赖根目录下的临时数据路径。
